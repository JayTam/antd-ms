# 设计原则

## 1、反馈

帮助用户了解系统或自己正在做什么，发现自己下一步可以做什么，以及明白动作会产生的结果。

- 过程反馈，在操作过程中，让用户对当前所处状态有所感知

<img src="/images/design/feedback1.png">

- 结果反馈，通过页面提示和交互动效让用户可以清晰的感知到自己的操作，即时的响应会给用户增加信赖感

<img src="/images/design/feedback2.png">

## 2、一致

将系统内元素统一，界面设计中遵循一致性能减少用户认知负担和记忆负担。

- 与现实一致，使用用户熟悉的语言、短语、术语，遵循现实世界的思维逻辑，避免使用生僻的系统语言。

<img src="/images/design/consistency1.png">

- 界面一致，界面的色彩一致、操作一致。

<img src="/images/design/consistency2.png">

## 3、效率

简单直接、清晰明确、帮助用户识别；界面设计中遵循效率原则可以提升产品的易用性。

- 化繁为简，让操作流程、交互更简单

<img src="/images/design/efficiency1.png">

- 对齐：界面上每个元素之间应存在某种视觉联系，这样能建立清晰的视觉流向、引导用户阅读信息以及快速帮助用户进行判断。

<img src="/images/design/efficiency2.png">

- 亲密性：相关性的项应靠近，归为一类。有助于信息呈现，提供清晰的结构。

<img src="/images/design/efficiency3.png">

## 4、容错性

容错性是产品对错误操作的承载性能，即能提升产品纠错效率，也可降低用户操作出错概率。

- 及时有效的提示，当用户进行重大决策、不可逆操作时，系统需要告知在用户做出此操作后将产生的后果，让用户自主选择是否继续执行。

<img src="/images/design/fault1.png">

- 帮助用户从错误中恢复，迅速回到正确状态。

<img src="/images/design/fault2.png">
