# ADR-0004: 子包独立性确认——三包均独立对外

- 状态: Accepted
- 日期: 2026-07-21
- 决策者: JayTam

## 背景

ADR-0003 选定洋葱式发布的前提是：`@jaytam/icons`、`@jaytam/request-ms`、`@jaytam/schema-render` 三个子包应当独立存在于 npm 上。本轮需确认这一前提的合法性。

## 选项

- **A1.** 三包都有外部消费方 / 对外发布计划，A 路径完全成立。
- **A2.** 只有 antd-ms 消费，应改选 B（内联进主包产物）。
- **A3.** 混合：部分子包独立、部分仅服侍 antd-ms。

## 决策

采用 **A1**：三个子包均作为独立 npm 包长期存在，各自走自己的 SemVer。

## 理由

- 三包均有外部消费方或对外发布计划，独立发布可让其它项目按需引用。
- 维持 monorepo 既有的 `packages/*` 边界，不引入"内联编译"这种非标操作。

## 后果

- 三个子包各自承担 npm 维护责任：CHANGELOG、版本对齐、deprecate 政策。
- 任何子包发版不可破坏既有 API（除非走 major）。
- 主包 `antd-ms` 的 deps 必须与子包实际发布版本对齐，见 ADR-0005。
